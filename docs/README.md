# docs/

Project documentation. Treated as code: every doc has an owner (see
`CODEOWNERS`), every change goes through review, and PR descriptions link back
to the relevant doc when behaviour changes.

| Folder | Purpose |
|---|---|
| `adr/` | Architecture Decision Records. One file per decision (`NNNN-title.md`). |
| `architecture/` | System design notes, diagrams, data flow. |
| `pitch/` | Bootcamp pitch deck (6 slides) + 3-minute demo script (Week 5 deliverable). |
| `benchmarks/` | Intel measurement methodology + raw runs. Reproducible numbers only. |
| `legacy/` | Pre-pivot CareerOS material kept for reference. Not the source of truth for current architecture. |
